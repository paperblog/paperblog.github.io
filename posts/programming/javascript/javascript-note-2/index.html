async vs defer#

1. head 안에 그냥 js 삽입하는 경우

<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
    <script src="main.js"></script>
</head>

단점

- 중간에 HTML 파싱이 멈출 수 있음

2. body 맨 끝에 js 삽입하는 경우

</body>
    <div></div>
    <script src="main.js"></script>
</html>

단점

- 자바스크립트에 의존적인 콘텐츠인 경우 사용자가 정상적인 페이지를 보기 전까지 대기시간이 길어질 수 있음

3. head 안에 넣되, async 속성 값을 주는 경우

<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
    <script async src="main.js"></script>
</head>

장점

- body 끝에 사용하는 경우와 비교하면 fetching이 HTML을 파싱하는 것과 병렬적으로 실행되므로 시간을 절약할 수 있음

단점

- HTML 파싱 전 자바스크립트가 실행되는 동안 대기시간이 길어질 수 있음

- 순서에 의존적이라면 문제가 될 수 있음

4. head 안에 넣되, defer 속성 값을 주는 경우

<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
    <script defer src="main.js"></script>
</head>

장점

- HTML 파싱 먼저 후 js 실행할 수 있음

- 순서에 의존적인 경우에도 잘 작동함

use strict를 선언하자#

- javascript 코드 작성 전 가장 상단에 아래와 같이 use strict를 정의하는 것을 권장

'use strict';

Why?

- 선언하기 전에 변수에 값을 할당하는 등의 비상식적인 로직을 막아줄 수 있음

// 'use strict';
a = 1;

- use strict 선언 시 비상식적인 로직에 대해 오류를 출력해줌

'use strict';
a = 1;

- 이 오류를 없애려면 아래와 같이 let을 통해 변수를 먼저 정의해주면 됨

'use strict';
let a;
a = 1;
